## Decisions Made

- **Untracked `data/` from git (important).** The clinical data was not merely
  in history — the 17 files under `data/` were *actively tracked* in the
  current tree, so `.gitignore` alone would not have kept them out of future
  commits. Ran `git rm --cached -r data/`, which stops tracking them while
  leaving every file on disk for the PI's local use (per the "do not delete
  data" constraint). The commit is pure deletion (336,598 line deletions, no
  PHI content added).
- **`miceforest` pinned to `>=6.0.0` and the MICE call adapted to the 6.x API.**
  The research code targeted 5.x (`datasets=` kwarg), but miceforest 5.x calls
  `np.Inf`, removed in NumPy 2.0, so it cannot run in a modern environment.
  Pinning NumPy `<2` would break the `deep`/`llm` extras (torch/TF/transformers).
  I kept the same effective configuration (1 dataset, 5 MICE iterations, 80
  LightGBM boosting rounds, max_depth 10) and documented the change inline. See
  blocker below re: exact paper reproducibility.
- **Added first-class `mean`/`median` methods.** CLAUDE.md lists "mean, median"
  among the comparison baselines, but in the notebooks they appeared only as
  fallback paths. I exposed them as proper `apply_*_imputation` functions with
  the same signature + validation reporting so they can be benchmarked
  uniformly. (`simple.py`.)
- **`columns.py` encodes FACT-E item/subscale/visit structure.** This is public
  questionnaire metadata (column codes, visit schedule, 0–4 Likert range), not
  patient data, so it is safe to commit and is needed by both the synthetic
  generator and downstream methods.
- **`src/` layout** (not flat) to keep the importable package cleanly separated
  from the notebooks/scripts that remain in the repo root.
- **Preserved research logic verbatim.** Imputation algorithms were copied
  exactly; only imports/wiring changed (plus the documented MICE API shim).

## Blockers / Questions for PI

1. **CRITICAL — PHI in notebook outputs.** 35 `.ipynb` files are tracked and
   very likely contain rendered cell outputs with real patient data (dataframe
   previews, value counts, per-patient tables). Before the repo can go public
   these outputs must be stripped (e.g. `nbstripout`) or the notebooks excluded.
   I did **not** strip them autonomously because that is destructive and could
   remove research results you want to keep. **Decision needed:** strip outputs
   in place, move notebooks to a private location, or exclude them from the
   public repo?
2. **History rewrite still required.** As noted in CLAUDE.md, `data/` remains in
   git history (and now the notebook outputs do too). The public release needs a
   history scrub. Confirming this is a separate, PI-owned step.
3. **MICE reproducibility.** Exact numerical reproduction of the *paper's* MICE
   baseline requires miceforest 5.x + NumPy <2 (a pinned repro environment).
   The packaged version runs on miceforest 6.x. Do you want a separate
   `requirements-repro.txt` / locked env documented for paper reproduction?
4. **`result plot/` directory** (many PNGs) is tracked. These look like
   aggregate plots (architectures, correlation/distribution summaries) and are
   probably fine, but please confirm none embed individual-level PHI.
5. **License holder.** Used `MIT` with copyright "Crump Lab" / 2026. Confirm the
   correct legal name / whether a specific author list should be listed.
