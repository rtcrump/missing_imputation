# missing_imputation

Imputation of missing values in **longitudinal clinical patient-reported outcome (PRO) data**, built around the FACT-E (Functional Assessment of Cancer Therapy — Esophageal) questionnaire.

This package provides a uniform interface to impute missing values in ordinal clinical questionnaire data and to **benchmark** imputation methods against held-out values, so you can pick the method that best preserves your data.

!!! info "v1.0 — Classical methods"
    The current release includes five classical imputation methods (mean, median, KNN, MICE, SoftImpute) and a full evaluation suite. Deep-learning and LLM-based methods are planned for v1.1.

## Key features

- **Five imputation methods** with a uniform API — swap methods with a single argument change
- **Evaluation suite** — cross-validated benchmarking with ordinal-aware metrics (QWK, within-1-category accuracy)
- **Longitudinal evaluators** — trajectory fidelity, temporal smoothness, and missingness-pattern robustness
- **Synthetic data generator** — PHI-free FACT-E data for testing and examples
- **CLI** — impute CSV files and generate demo data from the command line
- **No GPU required** — the core install has no deep-learning dependencies

## Quick example

```python
import missing_imputation as mi
from missing_imputation.columns import FACTE_COLUMNS

# Generate PHI-free synthetic FACT-E data
df = mi.make_synthetic_facte(n_patients=80, n_visits=5, missing_rate=0.2)

# Impute with KNN
imputed, _ = mi.apply_knn_imputation(df, FACTE_COLUMNS)

assert imputed[FACTE_COLUMNS].isna().sum().sum() == 0
```

## Data safety

This repository contains **no patient data**. All examples and tests run on synthetic data generated by `make_synthetic_facte`, which reproduces only the *structure* of the data (column names, visit schedule, ordinal range), never any real values.
